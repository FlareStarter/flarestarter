## Cron Triggers（定时任务）

CF 的定时任务通过 worker 的 `scheduled` handler 实现,而非 `fetch`。本模板已接好一个参考实现:

- **调度**:`wrangler.jsonc` 的 `triggers.crons`(默认每日 03:00 UTC)。命名环境 staging/production 不继承顶层配置,各自重复声明(与 bindings 同理)。
- **入口**:`src/worker.ts` 的 `scheduled()`(与 `fetch` 并列,一同被 `Sentry.withSentry` 包裹)。
- **逻辑**:`src/features/maintenance/cleanup.ts` 的 `runCleanup()` —— 删过期 session / verification token + 陈旧 better-auth 限流行。`now` 注入以便测试(`cleanup.workers.test.ts`)。
- **扩展**:把你自己的周期任务(发摘要邮件、按天清理 [配额计数](/docs/platform/security)等)加进 `scheduled()` 或 `runCleanup()` 即可。

本地手动触发(无需等到点):

```bash
pnpm dev                          # 另开一个终端
curl "http://localhost:3000/cdn-cgi/handler/scheduled"   # miniflare 暴露的 scheduled 触发端点
```

> 注意:Cron Triggers 在 CF **免费档可用**(零成本);Queues 才需付费档。所以"定时轮询"类需求用 cron,无需 Queues。
